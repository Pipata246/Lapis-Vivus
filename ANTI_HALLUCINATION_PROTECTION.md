# 🛡️ Анти-галлюцинационная защита v26.9

## Проблема
ИИ может "галлюцинировать" и выполнять несколько блоков сразу или переходить к следующему блоку самостоятельно, игнорируя команду сервера.

## Решение - 5 уровней защиты

### 1️⃣ Уровень: Системный промпт (`src/prompts/lapis-system.txt`)

**Что делает:**
- Явно запрещает объединение блоков
- Указывает единственный допустимый формат ответа
- Объясняет правило "следующий_блок" (информация, НЕ команда)

**Ключевые правила:**
```
⛔️ КРИТИЧЕСКОЕ ОГРАНИЧЕНИЕ: ОДИН ANSWER = ОДИН БЛОК
- Объединять несколько блоков в один answer - ЗАПРЕЩЕНО
- Выполнять блоки "про запас" - ЗАПРЕЩЕНО
- Упоминать блоки кроме текущего - ЗАПРЕЩЕНО

⚠️ ПРАВИЛО КОМАНДЫ СЕРВЕРА:
- Сервер указывает: "сервер_назначил_блок": "X" и "следующий_блок": "Y"
- Ты выполняешь ТОЛЬКО блок X
- Блок Y — информация для тебя, НЕ команда
```

### 2️⃣ Уровень: Команда оператора (`blockRunner.js` → `buildBlockMandate`)

**Что делает:**
- Создает жесткую визуальную рамку с командой
- Явно указывает запрещенные блоки
- Повторяет допустимый формат ответа

**Пример команды:**
```
═══════════════════════════════════════════════════════════════
⚠️ АБСОЛЮТНАЯ КОМАНДА ОПЕРАТОРА (сервер жёстко фиксирует шаг)
═══════════════════════════════════════════════════════════════

ШАГ 5 ИЗ 32
ЕДИНСТВЕННЫЙ АКТИВНЫЙ БЛОК: 1E

───────────────────────────────────────────────────────────────
ЗАДАНИЕ: БЛОК 1E: СОМАТИЧЕСКАЯ ИНТЕГРАЦИЯ ТРАВМЫ
───────────────────────────────────────────────────────────────

⛔️ КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО В ЭТОМ ANSWER:
  • Выполнять блоки: 1A, 1B, 1C, 1D, 2A, 2B, ... (все кроме 1E)
  • Создавать JSON-артефакты других блоков
  • Объединять несколько блоков

✅ ЕДИНСТВЕННЫЙ ДОПУСТИМЫЙ ФОРМАТ ANSWER:
  1. JSON-артефакт блока 1E
  2. Раздел ## Метакомментарии_Блока
  3. БОЛЬШЕ НИЧЕГО
```

### 3️⃣ Уровень: Payload оператора (`blockRunner.js` → `buildOperatorPayload`)

**Что делает:**
- Передает текущий блок: `сервер_назначил_блок`
- Передает следующий блок: `следующий_блок` (для информации!)
- Явно перечисляет запрещенные блоки
- Повторяет инструкцию в `инструкция_исполнения`

**Структура:**
```json
{
  "сервер_назначил_блок": "1E",
  "следующий_блок": "2A",
  "следующий_блок_описание": "БЛОК 2A: АРХИТЕКТУРА У-СИН",
  "запрещённые_блоки_в_этом_answer": ["1A", "1B", ..., "2A", ...],
  "инструкция_исполнения": "⚠️ СТРОГО: Выполни ТОЛЬКО БЛОК 1E. ЗАПРЕЩЕНО выполнять 2A в этом answer."
}
```

### 4️⃣ Уровень: Валидация ответа (`validateResponse.js`)

**Что делает:**
- Проверяет что в ответе только ОДИН JSON-артефакт
- Проверяет что JSON соответствует текущему блоку
- Проверяет наличие метакомментариев
- Отклоняет ответ если найдены лишние блоки

**Проверки:**
```javascript
// 1. Проверка JSON-артефакта
if (!answer.includes(jsonArtifactName(blockId))) {
  issues.push('Отсутствует JSON-артефакт текущего блока');
}

// 2. Проверка на лишние блоки
const otherBlocks = BLOCK_IDS.filter(id => id !== blockId);
for (const otherId of otherBlocks) {
  if (answer.includes(jsonArtifactName(otherId))) {
    issues.push(`Найден JSON-артефакт блока ${otherId}`);
  }
}

// 3. Проверка метакомментариев
if (!answer.includes('## Метакомментарии_Блока')) {
  issues.push('Отсутствуют метакомментарии');
}
```

### 5️⃣ Уровень: Retry с усиленной командой

**Что делает:**
- Если валидация не прошла - отправляет retry-запрос
- В retry явно указывает что было не так
- Повторяет команду с усилением

**Пример retry:**
```
Ответ отклонён (Найден JSON-артефакт блока 2A; Лишний контент).
Перегенерируй ТОЛЬКО блок 1E: БЛОК 1E: СОМАТИЧЕСКАЯ ИНТЕГРАЦИЯ ТРАВМЫ
JSON: блок_1E_инвариантСтрогийЗапуск_v26.9.json
## Метакомментарии_Блока
Не используй заголовки других блоков.
```

## Как это работает вместе

```
Пользователь нажимает "Запустить блок"
    ↓
Сервер определяет текущий блок (например, 1E)
    ↓
buildBlockMandate() создает жесткую команду
    ↓
buildOperatorPayload() добавляет:
  - текущий блок: 1E
  - следующий блок: 2A (для информации!)
  - запрещенные блоки: все кроме 1E
    ↓
Отправка в ИИ с системным промптом
    ↓
ИИ видит:
  - Системный промпт: "ОДИН БЛОК = ОДИН ANSWER"
  - Команду: "ТОЛЬКО БЛОК 1E"
  - Payload: "следующий_блок: 2A (НЕ выполнять!)"
    ↓
ИИ генерирует ответ
    ↓
validateBlockResponse() проверяет:
  ✓ Есть JSON блока 1E?
  ✓ Нет JSON других блоков?
  ✓ Есть метакомментарии?
    ↓
Если НЕТ → Retry с усиленной командой
Если ДА → Сохранение и переход к следующему блоку
```

## Почему это работает

1. **Многократное повторение** - правило повторяется 5 раз в разных формах
2. **Визуальные якоря** - рамки, эмодзи, заглавные буквы привлекают внимание
3. **Явные запреты** - перечисляем ВСЕ запрещенные блоки
4. **Информация о следующем** - ИИ знает что будет дальше, но НЕ выполняет
5. **Валидация + Retry** - если ИИ ошибся, даем второй шанс с усилением
6. **Контекст сессии** - ИИ видит только текущую сессию, не путается в истории

## Мониторинг

Для проверки эффективности защиты смотрим:
- Логи `validateBlockResponse()` - сколько ответов отклонено
- Причины отклонения - какие именно блоки "просочились"
- Количество retry - как часто нужен второй запрос
